voxel.js

by J. Brian Shumate

In the last several months, I’ve been playing around with voxel.js. It’s been great fun with my kids, and useful for helping to teach my daughter programming with JavaScript.

Both kids love it for its similarities to Minecraft, and I appreciate its open source nature and the ability to do things in the browser by default.

If you haven’t yet checked out voxel.js, I highly encourage you to do so; it was built and is maintained by some really cool people, and is a ton of fun to play with.

The project was built with the Node.js JavaScript platform, is heavily reliant on three.js, and is relatively simple to get up and running on your own computer or on a server near you.

Moving around in the voxel.js world relies on web browser support for mouse lock (also called pointer lock) which is supported in contemporary versions of Chrome and Firefox at least. I have not personally it tried with other web browsers.

There are numerous modules which can be used to extend the basic functionality supplied with voxel.js so that you can introduce characters (critters), change physics, play audio, interact with objects, and much more.

There’s even a Voxel Critter Creator, which you can use to design different critters for use with voxel.js, and it is pretty simple to import existing Minecraft skins as well.

A neat little red couch voxel “critter”, ready to be used in a voxel world or NoSQL database use case near you ;)

All in all, voxel.js is probably one of the coolest JavaScript and game focused projects going for introducing novice programmers to more advanced concepts while having a fun time.

Check it out and don’t forget to marvel at the sheer magnificence that is Max Ogden’s beard!
